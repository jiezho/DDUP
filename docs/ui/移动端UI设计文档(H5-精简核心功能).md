# 移动端 UI 设计文档（H5｜精简核心功能）

## 0. 目标与定位
- 定位：移动端以“随手用”为主，强调低认知负担、单手可达、快速记录与快速查看。
- 原则：只保留核心闭环（对话→行动、学习→复习、助手→执行、我的→复盘/配置）；重型模块（资源/文件/图谱/模板市场）在移动端弱化为入口或二级页。
- 视觉：更轻、更干净；对比 PC 端的“工作台质感 + 科技感”，移动端更偏“系统原生 + 可读性优先”。

## 1. 信息架构（H5）
### 1.1 一级导航（底部 TabBar）
- 对话：对话输入与结果卡（支持写入 Wiki 暂存区）
- 学习：复习队列 + 术语库
- 助手：待办 / 习惯 / 灵感
- 我的：统计 / 模板 / 集成 / 权限 / Wiki 状态 / 更多入口

说明：原型包含“资源”一级入口，但移动端默认不常驻（避免信息密度与任务切换成本）。如需访问资源类能力，通过“我的 → 更多”进入二级页或使用对话触发。

### 1.2 二级入口（从“我的”或对话卡进入）
- 资源聚合页（资讯/图谱/文件/Wiki 检索）
- 模板中心（预览/使用）
- 集成与连接（飞书/微信/多维表格）
- 权限与审计（只读状态）

## 2. 页面规格（H5）
### 2.1 对话（/home）
- 布局：单列；顶部轻量状态条；底部输入 Dock 固定。
- 交互：
  - 输入框固定底部，点击输入时随键盘上移，页面保持可滚动阅读历史。
  - 流式输出：逐字增长；遇到长内容时自动滚动只在“用户未手动上滑”情况下触发。
  - 结果卡：默认折叠到摘要；提供“写入 Wiki”与 action buttons，按钮必须可单手点击（最小高度 40px）。

### 2.2 学习（/learning）
- 复习队列：
  - 单卡单列：术语（强）+ 释义（次）
  - 评分操作固定在卡片底部区域，按钮间距 >= 8px
- 术语库：
  - 新增表单放在顶部，输入区可折叠（展开/收起），避免占用过多首屏空间
  - 列表项信息精简：term、掌握状态、下次复习（可选）

### 2.3 助手（/assistant）
- 待办：
  - 快速新增：单行输入 + 添加按钮
  - 列表项：完成/撤销为主；更多操作放入二级（长按/更多）
- 习惯：
  - 列表项突出“打卡”主动作；次要信息（cadence/streak）以小字展示
- 灵感：
  - 快速记录为核心：TextArea + 保存
  - 标签输入可选；默认不强制

### 2.4 我的（/me）
- 顶部信息卡：用户/模式提示/刷新入口（可选）
- Tabs 允许滚动；默认展示“统计仪表盘”
- 统计：优先 2x2 栅格或单列，保持可读性
- Wiki 状态：只展示“是否启用/Raw 数/最近更新时间”三类信息；其余细节放到“查看详情”
- 更多入口：以列表呈现二级能力（资源、模板中心、集成、权限）

## 3. 组件与视觉规范（H5）
### 3.1 栅格与间距
- 页面 padding：12
- 卡片间距：12
- 列表项间距：8

### 3.2 字体与层级
- 标题：16–18（加粗）
- 正文：14
- 辅助信息：12（灰度）

### 3.3 触控与可达性
- 主要按钮最小高度：40
- 交互控件点击热区：>= 40x40
- 避免需要 hover 的交互作为唯一入口

### 3.4 颜色策略
- 移动端保持浅色为主（白底 + 轻阴影），强调色用于主按钮与关键标签。
- 弱化大面积渐变与装饰背景，优先内容可读性。

## 4. 与 PC 端的差异（对齐原型但做取舍）
- PC：工作台式信息密度（更多模块入口、更完整的资源/知识管理视图）、更强的空间感与“科技感”背景。
- H5：核心闭环优先；资源/文件/图谱等重型能力下沉到二级入口或通过对话触发。
- 组件形态差异：
  - PC 列表/详情倾向双栏或侧边预览；H5 统一为单列 + Drawer/全屏详情。

## 5. 验收清单（H5）
- 底部导航 4 个入口均可达，且刷新后仍保持模式选择
- 对话输入固定底部，键盘弹出不遮挡输入框
- 结果卡按钮在移动端可点击且不溢出
- 学习/助手/我的页面在 360px 宽度下可正常使用
