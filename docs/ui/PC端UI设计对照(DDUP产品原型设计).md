# PC 端 UI 设计对照（对齐《DDUP产品原型设计》）

## 0. 目标
- PC 端承担“工作台”：信息密度更高、入口更全、支持多任务切换与并行查看（列表 + 预览/侧栏）。
- 移动端承担“随手用”：功能精简、单列为主、复杂能力下沉二级入口。

## 1. 原型模块 → 当前实现（PC）
> 以一级导航为主线，优先对齐原型的页面结构与内容分区，不改变业务语义。

### 1.1 首页（对话 / 工作台 / 今日待处理 / 快速入口 / 仪表盘）
- 已实现：对话（流式输出 + 结果卡 + 写入 Wiki 暂存区）
- 待补齐（PC 优先）：
  - 工作台：快捷卡片（今日待处理、最近空间、最近知识卡、近期产出）
  - 仪表盘：关键指标入口（可复用“我的-统计”组件并在首页做摘要）

### 1.2 学习（路径 / 今日任务 / 术语库 / 卡片学习 / 复习队列 / 课程 / 学习笔记）
- 已实现：术语库、复习队列（MVP）
- 待补齐（PC 优先）：
  - 学习路径与今日任务：以“路线图 + 今日卡”形式落在页面顶部
  - 学习笔记：与 Wiki 的“正式页面”打通（查询/引用/回链）

### 1.3 助手（日历+清单 / 习惯打卡 / 灵感收件箱 / 详情 / 备忘录）
- 已实现：待办、习惯、灵感（MVP）
- 待补齐（PC 优先）：
  - 日历视图：周/月视图 + 待办关联（先做展示与筛选）
  - 灵感详情：右侧抽屉预览（PC）+ 标签/关联来源

### 1.4 资源（资讯 / 资讯源 / 稍后读 / 文件检索 / PDF 阅读 / 论文库 / 注释提取 / 图谱 / 智能体对话）
- 已实现：资讯源/信息流、图谱实体、文件列表（MVP）；Wiki 检索入口
- 待补齐（PC 优先）：
  - 文件检索：列表 + 右侧预览（原型强调 PC 侧预览）
  - 论文与 PDF：阅读器/标注/笔记（分阶段）
  - 稍后读/收藏：信息流的二级筛选视图

### 1.5 我的（偏好 / 统计复盘 / 模板中心 / 同步与存储 / 隐私 / 权限 / 审计 / 智能体中心 / 集成）
- 已实现：统计、模板（占位）、集成（占位）、权限（占位）、Wiki 状态
- 待补齐（PC 优先）：
  - 模板中心：模板市场式列表（卡片 + 标签 + 预览）
  - 集成：飞书/微信/多维表格联动的状态与配置页
  - 智能体中心：Hermes/Skills 管理入口（视权限显示）

## 2. PC vs H5 的明确差异（设计约束）
- 导航：PC（Sider + 顶部胶囊 Topbar + Hero），H5（底部 TabBar 精简 + 二级入口）
- 信息密度：PC 允许双栏/侧边预览；H5 统一单列 + Drawer/全屏详情
- 资源/文件/图谱：PC 主场景；H5 仅保留入口与轻量浏览

## 3. 迭代优先级（建议）
- P0：首页增加“今日待处理/快捷入口”分区；资源页增加 PC 预览布局
- P1：学习路径/今日任务；灵感详情抽屉
- P2：模板中心完整化；集成配置页
