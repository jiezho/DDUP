# PC 端页面设计规范（Workbench 质感 + 科技感）

## 0. 设计目标
- 参考：Power-Agent Workbench 的“圆角大卡片 + 胶囊导航 + 低对比边框 + 柔和投影 + 背景渐变”。
- 融入科技感：使用轻量光晕、细网格、微玻璃材质，不做重动效、不影响性能与可读性。
- 不改变业务：仅调整主题、布局密度与视觉层级。

## 1. 视觉语言
### 1.1 色彩与材质
- 背景：暖色浅底 + 多层柔和光晕（蓝/青/靛）
- 面板：半透明白（glass）+ 轻边框
- 强调色：#3B82F6（主按钮/高亮）；成功态辅色可用绿色

### 1.2 圆角与阴影
- Topbar：胶囊（radius 999）
- Hero/卡片：大圆角（16–20）
- 阴影：偏“柔”，避免强对比黑影（以 rgba(15,23,42,0.06–0.10) 为主）

### 1.3 排版与信息层级
- 采用“Hero（标题+一句话说明）→ 功能卡片”的层级结构
- 列表与表单避免堆叠分割线，优先通过间距与分组标题形成节奏

## 2. 布局规范（PC）
- 左侧 Sider：浅色半透明；菜单保持 5 个一级入口（首页/学习/助手/资源/我的）
- 顶部 Topbar：胶囊容器；右侧保留显示模式切换
- 页面 Hero：每个路由首屏提供标题与一句话说明，用于对齐“工作台”风格与增强留白
- 内容容器：max-width 1200，保持居中；复杂页面内部再做分区卡片

## 3. 交互细节
- Card hover：轻微上浮（1px）+ 阴影增强 + 边框高亮
- Button active：轻微下沉（1px）
- 过渡：120–220ms，缓动使用 cubic-bezier(0.22, 1, 0.36, 1)

## 4. 与移动端的区分
- PC：入口更全、信息密度更高、允许并行展示（列表+预览/侧栏）
- H5：核心闭环优先，复杂能力下沉二级入口（详见《移动端 UI 设计文档》）
