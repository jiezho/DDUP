# DDUP 部署文档（V0.1）

## 1. 本地开发部署

### 1.1 PostgreSQL（docker-compose）

目录：`infra/docker-compose.yml`

- 默认参数
  - DB：`ddup`
  - 用户/密码：`ddup/ddup`
  - 端口：`5432`

### 1.2 后端（FastAPI）

目录：`apps/api/`

- 安装依赖：`python -m pip install -r requirements.txt`
- 启动：`python -m uvicorn app.main:app --host 0.0.0.0 --port 8000`

环境变量（示例）：
- `DATABASE_URL=postgresql+psycopg://ddup:ddup@localhost:5432/ddup`

### 1.3 前端（React）

目录：`apps/web/`

- 安装依赖：`npm install`
- 启动：`npm run dev`

前端 dev server 会将 `/api/*` 与 `/healthz` 代理到 `http://localhost:8000`。

## 2. 生产部署建议（MVP）

### 2.1 组件清单

- Web：静态资源（建议 Nginx 托管）
- API：FastAPI（Uvicorn/Gunicorn）
- DB：PostgreSQL
- 可选对象存储：MinIO/S3（文件、PDF、图片）

## 3. 发布环境部署（Ubuntu/Debian + PostgreSQL 宿主机 + 应用 Docker/Compose）

目标环境：
- 服务器：`192.168.102.204`
- 代码仓库：`https://github.com/jiezho/DDUP.git`
- 部署方式：PostgreSQL 宿主机安装；API/Web 用 docker compose

### 3.1 安装基础服务（服务器上执行）

```bash
sudo apt update
sudo apt install -y postgresql postgresql-contrib docker.io docker-compose-plugin
sudo systemctl enable --now postgresql
sudo systemctl enable --now docker
sudo usermod -aG docker $USER
newgrp docker
docker version
docker compose version
```

如 `docker compose` 或镜像拉取失败（Docker Hub 超时），可配置镜像加速后重试：

```bash
sudo mkdir -p /etc/docker
cat >/etc/docker/daemon.json <<'JSON'
{
  "registry-mirrors": ["https://docker.m.daocloud.io"]
}
JSON
sudo systemctl restart docker
docker info | grep -i mirror
```

### 3.2 初始化 PostgreSQL（服务器上执行）

```bash
sudo -u postgres psql
```

在 psql 内执行（密码替换为强密码）：

```sql
CREATE ROLE ddup WITH LOGIN PASSWORD 'REPLACE_WITH_STRONG_PASSWORD';
CREATE DATABASE ddup OWNER ddup;
\c ddup
CREATE EXTENSION IF NOT EXISTS pg_trgm;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
\q
```

如 PostgreSQL 安装在宿主机、应用跑在容器中，需允许容器网段访问 PostgreSQL：

```bash
sudo sed -i -E "s/^[#[:space:]]*listen_addresses[[:space:]]*=.*/listen_addresses = '*'/" /etc/postgresql/16/main/postgresql.conf
echo "host ddup ddup 172.16.0.0/12 scram-sha-256" | sudo tee -a /etc/postgresql/16/main/pg_hba.conf
sudo systemctl restart postgresql
sudo ss -lntp | grep 5432
```

### 3.3 拉取代码与配置（服务器上执行）

```bash
mkdir -p ~/ddup
cd ~/ddup
git clone https://github.com/jiezho/DDUP.git .
cp .env.prod.example .env
```

编辑 `.env` 中的 `DATABASE_URL`，推荐使用：
- `host.docker.internal`（compose 已配置 `extra_hosts: host-gateway`）：
  - `postgresql+psycopg://ddup:<PASSWORD>@host.docker.internal:5432/ddup`

### 3.4 构建与启动（服务器上执行）

```bash
docker compose --env-file .env -f infra/docker-compose.prod.yml up -d --build
docker compose -f infra/docker-compose.prod.yml ps
```

对外端口：
- `80`：Web（Nginx 容器）
- `8000`：API（如只需要经 Web 代理访问，可在 compose 中移除 8000 端口映射）

### 3.5 健康检查（服务器或任意可访问机器）

```bash
curl -s http://127.0.0.1/ | head
curl -s http://127.0.0.1/healthz
curl -I http://127.0.0.1/docs
```

### 3.6 更新发布（服务器上执行）

```bash
cd ~/ddup
git pull
docker compose --env-file .env -f infra/docker-compose.prod.yml up -d --build
```

### 2.2 反向代理建议

- `web` 与 `api` 同域：
  - `/` → web 静态资源
  - `/api` 与 `/healthz` → 后端

### 2.3 安全与配置

- 生产环境必须：
  - 使用真实的用户认证（当前 `X-User-Id` 仅为开发占位）
  - 外部集成（飞书/微信）凭据不得落库明文，使用环境变量/密钥服务
  - 配置审计留存策略

## 4. 运维检查点

- 健康检查：`GET /healthz`
- Swagger：`GET /docs`
- OpenAPI：`GET /openapi.json`
