# DDUP 部署文档（V0.1）

## 1. 本地开发部署

### 1.1 PostgreSQL（docker-compose）

目录：`infra/docker-compose.yml`

- 默认参数
  - DB：`ddup`
  - 用户/密码：`ddup/ddup`
  - 端口：`5432`

### 1.2 后端（FastAPI）

目录：`apps/api/`

- 安装依赖：`python -m pip install -r requirements.txt`
- 启动：`python -m uvicorn app.main:app --host 0.0.0.0 --port 8000`

环境变量（示例）：
- `DATABASE_URL=postgresql+psycopg://ddup:ddup@localhost:5432/ddup`

### 1.3 前端（React）

目录：`apps/web/`

- 安装依赖：`npm install`
- 启动：`npm run dev`

前端 dev server 会将 `/api/*` 与 `/healthz` 代理到 `http://localhost:8000`。

## 2. 生产部署建议（MVP）

### 2.1 组件清单

- Web：静态资源（建议 Nginx 托管）
- API：FastAPI（Uvicorn/Gunicorn）
- DB：PostgreSQL
- 可选对象存储：MinIO/S3（文件、PDF、图片）

### 2.2 反向代理建议

- `web` 与 `api` 同域：
  - `/` → web 静态资源
  - `/api` 与 `/healthz` → 后端

### 2.3 安全与配置

- 生产环境必须：
  - 使用真实的用户认证（当前 `X-User-Id` 仅为开发占位）
  - 外部集成（飞书/微信）凭据不得落库明文，使用环境变量/密钥服务
  - 配置审计留存策略

## 3. 运维检查点

- 健康检查：`GET /healthz`
- Swagger：`GET /docs`
- OpenAPI：`GET /openapi.json`
