# DDUP 部署文档（V0.1）

## 1. 本地开发部署

### 1.1 PostgreSQL（docker-compose）

目录：`infra/docker-compose.yml`

- 默认参数
  - DB：`ddup`
  - 用户/密码：`ddup/ddup`
  - 端口：`5432`

### 1.2 后端（FastAPI）

目录：`apps/api/`

- 安装依赖：`python -m pip install -r requirements.txt`
- 启动：`python -m uvicorn app.main:app --host 0.0.0.0 --port 8000`

环境变量（示例）：
- `DATABASE_URL=postgresql+psycopg://ddup:ddup@localhost:5432/ddup`

### 1.3 前端（React）

目录：`apps/web/`

- 安装依赖：`npm install`
- 启动：`npm run dev`

前端 dev server 会将 `/api/*` 与 `/healthz` 代理到 `http://localhost:8000`。

## 2. 生产部署建议（MVP）

### 2.1 组件清单

- Web：静态资源（建议 Nginx 托管）
- API：FastAPI（Uvicorn/Gunicorn）
- DB：PostgreSQL
- 可选对象存储：MinIO/S3（文件、PDF、图片）

## 3. 发布环境部署（Ubuntu/Debian + PostgreSQL 宿主机 + 应用 Docker/Compose）

目标环境：
- 服务器：`192.168.102.204`
- 代码仓库：`https://github.com/jiezho/DDUP.git`
- 部署方式：PostgreSQL 宿主机安装；API/Web 用 docker compose

### 3.1 安装基础服务（服务器上执行）

```bash
sudo apt update
sudo apt install -y postgresql postgresql-contrib docker.io docker-compose-plugin
sudo systemctl enable --now postgresql
sudo systemctl enable --now docker
sudo usermod -aG docker $USER
newgrp docker
docker version
docker compose version
```

如 `docker compose` 或镜像拉取失败（Docker Hub 超时），可配置镜像加速后重试：

```bash
sudo mkdir -p /etc/docker
cat >/etc/docker/daemon.json <<'JSON'
{
  "registry-mirrors": ["https://docker.m.daocloud.io"]
}
JSON
sudo systemctl restart docker
docker info | grep -i mirror
```

### 3.2 初始化 PostgreSQL（服务器上执行）

```bash
sudo -u postgres psql
```

在 psql 内执行（密码替换为强密码）：

```sql
CREATE ROLE ddup WITH LOGIN PASSWORD 'REPLACE_WITH_STRONG_PASSWORD';
CREATE DATABASE ddup OWNER ddup;
\c ddup
CREATE EXTENSION IF NOT EXISTS pg_trgm;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
\q
```

如 PostgreSQL 安装在宿主机、应用跑在容器中，需允许容器网段访问 PostgreSQL：

```bash
sudo sed -i -E "s/^[#[:space:]]*listen_addresses[[:space:]]*=.*/listen_addresses = '*'/" /etc/postgresql/16/main/postgresql.conf
echo "host ddup ddup 172.16.0.0/12 scram-sha-256" | sudo tee -a /etc/postgresql/16/main/pg_hba.conf
sudo systemctl restart postgresql
sudo ss -lntp | grep 5432
```

### 3.3 拉取代码与配置（服务器上执行）

```bash
mkdir -p ~/ddup
cd ~/ddup
git clone https://github.com/jiezho/DDUP.git .
cp .env.prod.example .env
```

编辑 `.env` 中的 `DATABASE_URL`，推荐使用：
- `host.docker.internal`（compose 已配置 `extra_hosts: host-gateway`）：
  - `postgresql+psycopg://ddup:<PASSWORD>@host.docker.internal:5432/ddup`

### 3.4 构建与启动（服务器上执行）

```bash
docker compose --env-file .env -f infra/docker-compose.prod.yml up -d --build
docker compose -f infra/docker-compose.prod.yml ps
```

对外端口：
- `80`：Web（Nginx 容器）
- `8000`：API（如只需要经 Web 代理访问，可在 compose 中移除 8000 端口映射）

### 3.5 健康检查（服务器或任意可访问机器）

```bash
curl -s http://127.0.0.1/ | head
curl -s http://127.0.0.1/healthz
curl -I http://127.0.0.1/docs
```

### 3.6 更新发布（服务器上执行）

```bash
cd ~/ddup
git pull
docker compose --env-file .env -f infra/docker-compose.prod.yml up -d --build
```

### 3.7 Hermes Agent（智能体）部署（服务器上执行）

说明：Hermes Agent 需要类 Unix 环境；本发布环境为 Ubuntu，可直接安装。建议使用独立用户运行，避免与 DDUP 容器权限混用。

#### 3.7.1 创建运行用户

```bash
sudo useradd -m -d /opt/hermes -s /bin/bash hermes || true
sudo ls -la /opt/hermes
```

#### 3.7.2 安装 Hermes（非交互）

官方一键安装脚本会在用户目录下创建 `~/.hermes/`，并安装 `hermes` 命令。

```bash
sudo -iu hermes bash -lc '
  mkdir -p /opt/hermes/tmp
  curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh -o /opt/hermes/tmp/hermes_install.sh
  bash /opt/hermes/tmp/hermes_install.sh --skip-setup
'
```

验证安装：

```bash
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes version'
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes doctor || true'
```

可选系统依赖（提升搜索与语音工具能力）：

```bash
sudo apt-get update -y
sudo apt-get install -y ripgrep ffmpeg
```

#### 3.7.3 配置模型与密钥

Hermes 的密钥与部分配置默认存放在：
- `/opt/hermes/.hermes/.env`

配置方式：

```bash
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes setup'
```

或使用命令写入配置（会写入正确的文件位置）：

```bash
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes config set OPENROUTER_API_KEY <YOUR_KEY>'
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes model'
```

##### 3.7.3.1 配置 GPUStack（OpenAI 兼容接口）

你提供的 GPUStack 接口为 OpenAI 兼容的 `chat/completions`，Hermes 建议使用 `custom` provider：

- `model.provider = custom`
- `model.base_url = http://192.168.102.201/v1`（注意：不要填到 `/v1/chat/completions`，base_url 应该是 `/v1`）
- `model.default = glm-5.1-fp8`
- 推理密钥：`OPENAI_API_KEY`（写入 `/opt/hermes/.hermes/.env`）

在服务器上执行（不会把密钥写入 `config.yaml`，只写入 `.env`）：

```bash
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes config set model.provider custom'
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes config set model.base_url http://192.168.102.201/v1'
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes config set model.default glm-5.1-fp8'
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes config set OPENAI_API_KEY <GPUSTACK_API_KEY>'
```

应用配置后重启 gateway：

```bash
sudo systemctl restart hermes-gateway.service
sudo /opt/hermes/.local/bin/hermes gateway status
```

#### 3.7.3.2 生产环境安全处理（强制）

- 密钥只放在服务器：`/opt/hermes/.hermes/.env`，不要提交到 git，不要写入镜像。
- 权限收敛：

```bash
sudo chmod 700 /opt/hermes/.hermes
sudo chmod 600 /opt/hermes/.hermes/.env /opt/hermes/.hermes/config.yaml
```

- 轮换策略：如需更换密钥，重复执行 `hermes config set OPENAI_API_KEY ...` 后重启 `hermes-gateway.service`。

#### 3.7.4 以 systemd 运行 Gateway（推荐）

Gateway 进程负责消息平台接入（Telegram/Discord 等）与定时任务调度。即使暂未配置平台，也建议先安装为系统服务，后续配置后直接生效。

安装 systemd 服务（运行用户为 `hermes`）：

```bash
sudo /opt/hermes/.local/bin/hermes gateway install --system --run-as-user hermes --force
```

启动与查看状态：

```bash
sudo /opt/hermes/.local/bin/hermes gateway start
sudo /opt/hermes/.local/bin/hermes gateway status
sudo journalctl -u hermes-gateway -f
```

#### 3.7.4.1 启用 Hermes API Server（供 DDUP 调用）

DDUP 后端会通过 Hermes 的 OpenAI 兼容接口调用 Hermes（`/v1/chat/completions`），用于：
- 对话流真正由 Hermes 生成（替换 DDUP 的 Mock 回声）
- Wiki 检索由 Hermes 调用 `wiki-query` 技能完成

1）在 Hermes 侧开启 API Server（写入 `/opt/hermes/.hermes/.env`，不要提交到 git）：

```bash
sudo -iu hermes bash -lc 'cat >> /opt/hermes/.hermes/.env << EOF
API_SERVER_ENABLED=true
API_SERVER_HOST=0.0.0.0
API_SERVER_PORT=8642
API_SERVER_KEY=<CHANGE_ME>
EOF
chmod 600 /opt/hermes/.hermes/.env'
```

2）重启 gateway：

```bash
sudo systemctl restart hermes-gateway.service
sudo /opt/hermes/.local/bin/hermes gateway status
```

3）在 DDUP 侧配置 Hermes 调用信息（写入 `/opt/ddup/.env`，供 `infra/docker-compose.prod.yml` 注入）：

```bash
HERMES_API_BASE=http://host.docker.internal:8642/v1
HERMES_API_KEY=<SAME_AS_API_SERVER_KEY>
HERMES_MODEL=hermes-agent
```

安全要求：
- 不要对公网暴露 8642；仅允许 DDUP 容器所在网段访问。
- 如需进一步收敛，可把 `API_SERVER_HOST` 绑定到 docker bridge 网卡地址，并在防火墙仅放行该网段。

如需配置消息平台：

```bash
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes gateway setup'
```

#### 3.7.5 更新/卸载

```bash
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes update'
sudo /opt/hermes/.local/bin/hermes gateway stop
sudo /opt/hermes/.local/bin/hermes gateway uninstall --system || true
```

#### 3.7.6 Obsidian Wiki（Vault）与 obsidian-wiki 技能（推荐）

用途：把 DDUP/Hermes 的对话、复盘、项目沉淀编译为可浏览的 Obsidian Wiki（Markdown + wikilinks），并通过增量方式持续更新。

1）创建 Vault 目录（hermes 负责编译写入；DDUP 负责写入 `_raw/`）：

```bash
sudo groupadd -f ddupwiki
sudo usermod -aG ddupwiki hermes

sudo mkdir -p /opt/ddup/wiki-vault/_raw
sudo chown -R hermes:ddupwiki /opt/ddup/wiki-vault
sudo chmod 2775 /opt/ddup/wiki-vault /opt/ddup/wiki-vault/_raw
```

2）在 `/opt/ddup/.env` 增加 DDUP Wiki 配置（供 `infra/docker-compose.prod.yml` 使用）：

```bash
DDUP_WIKI_ENABLED=true
DDUP_WIKI_VAULT_PATH=/opt/ddup/wiki-vault
DDUP_WIKI_RAW_DIR=_raw
```

3）准备 obsidian-wiki 仓库（供技能与配置引用）：

方式 A（服务器可访问 GitHub）：

```bash
sudo -iu hermes bash -lc 'cd /opt/hermes && git clone https://github.com/Ar9av/obsidian-wiki.git || true'
sudo -iu hermes bash -lc 'cd /opt/hermes/obsidian-wiki && git pull'
```

方式 B（服务器无法访问 GitHub，离线上传）：

```bash
# 在本地机器打包（排除 .git）后上传到服务器：
# tar -czf obsidian-wiki.tgz --exclude=.git -C <本地路径> obsidian-wiki
# scp obsidian-wiki.tgz root@<server>:/opt/hermes/obsidian-wiki.tgz

sudo rm -rf /opt/hermes/obsidian-wiki
sudo tar -xzf /opt/hermes/obsidian-wiki.tgz -C /opt/hermes
sudo rm -f /opt/hermes/obsidian-wiki.tgz
sudo chown -R hermes:hermes /opt/hermes/obsidian-wiki
```

4）把 obsidian-wiki 的技能注册到 Hermes（软链到 `~/.hermes/skills/`）：

```bash
sudo -iu hermes bash -lc '
  mkdir -p /opt/hermes/.hermes/skills
  for d in /opt/hermes/obsidian-wiki/.skills/*; do
    name=$(basename "$d")
    ln -sfn "$d" "/opt/hermes/.hermes/skills/$name"
  done
'
```

5）写入全局配置 `~/.obsidian-wiki/config`：

```bash
sudo -iu hermes bash -lc '
  mkdir -p /opt/hermes/.obsidian-wiki
  cat > /opt/hermes/.obsidian-wiki/config <<EOF
OBSIDIAN_VAULT_PATH=/opt/ddup/wiki-vault
OBSIDIAN_WIKI_REPO=/opt/hermes/obsidian-wiki
EOF
  chmod 600 /opt/hermes/.obsidian-wiki/config
'
```

6）手动验证（首次会初始化 Vault 结构）：

```bash
sudo -iu hermes bash -lc 'export PATH=/opt/hermes/.local/bin:$PATH; hermes "set up my wiki"'
```

后续可在 Hermes 中使用（命令名以技能为准）：
- `/wiki-update`：把当前项目沉淀到 Vault 的 `projects/`
- `/wiki-ingest`：处理 Vault `_raw/` 暂存
- `/wiki-query`：从 Wiki 回答问题
- `/wiki-lint`：健康检查
- `/wiki-export`：导出图谱（JSON/GraphML/Neo4j 等）

7）配置 Hermes 定时维护（推荐，每 60m 跑一次）：

说明：`--script` 仅支持 `~/.hermes/scripts/` 下的相对路径。

```bash
sudo -iu hermes bash -lc '
  mkdir -p /opt/hermes/.hermes/scripts
  cat > /opt/hermes/.hermes/scripts/wiki_maintenance_prompt.py <<PY
def main():
  print(\"Maintain DDUP Obsidian wiki vault: ingest _raw, cross-link, lint, update.\")

if __name__ == \"__main__\":
  main()
PY
  chmod 700 /opt/hermes/.hermes/scripts/wiki_maintenance_prompt.py
'

sudo -iu hermes bash -lc '
  export PATH=/opt/hermes/.local/bin:$PATH
  hermes cron create 60m --repeat 999999 --name ddup-wiki-maintain \
    --skill wiki-ingest --skill cross-linker --skill wiki-lint --skill wiki-update \
    --script wiki_maintenance_prompt.py \"\"
  hermes cron list
'
```

### 3.8 反向代理建议

- `web` 与 `api` 同域：
  - `/` → web 静态资源
  - `/api` 与 `/healthz` → 后端

### 3.9 安全与配置

- 生产环境必须：
  - 使用真实的用户认证（当前 `X-User-Id` 仅为开发占位）
  - 外部集成（飞书/微信）凭据不得落库明文，使用环境变量/密钥服务
  - 配置审计留存策略
