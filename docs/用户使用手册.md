# DDUP 用户使用手册（V0.1）

## 1. 产品概览

DDUP 是“AI个人成长平台”，默认以首页对话作为入口，通过对话调用各模块能力并返回可落地的结果卡。

当前版本（V0.1）已具备：
- 空间（个人/项目/团队）最小能力：自动创建个人空间、可切换空间
- 对话（最小闭环）：输入问题→流式返回文本与结构化卡片（MVP 占位）
- 审计：记录关键操作（对话、动作等）并可查询

## 2. 运行方式（本地）

### 2.1 启动数据库（可选）

如果你需要运行持久化数据（PostgreSQL）：
- 进入 `infra/`，使用 docker-compose 启动 postgres
- 默认数据库：`ddup`，用户/密码：`ddup/ddup`，端口：`5432`

### 2.2 启动后端

进入 `apps/api/`：
- 安装依赖：`python -m pip install -r requirements.txt`
- 启动服务：`python -m uvicorn app.main:app --reload --port 8000`

### 2.3 启动前端

进入 `apps/web/`：
- 安装依赖：`npm install`
- 启动开发服务：`npm run dev`

默认前端端口：`5173`。前端已配置代理，会把 `/api/*` 与 `/healthz` 转发到后端 `8000`。

## 3. 使用说明

### 3.1 首页对话

- 入口：底部导航“首页”
- 能力：
  - 输入问题后，系统会以流式方式返回回复
  - 当前阶段会返回一个最小“分析卡（analysis）”用于验证卡片链路

### 3.2 空间切换

- 首页顶部右侧有空间选择器
- 规则：
  - 如果你第一次访问，系统会为当前用户自动创建“个人空间”
  - 后续可创建“项目空间”（API 层已提供创建接口；前端页面后续补齐）

### 3.3 审计查询

用于追溯“谁在什么空间做了什么操作”。当前提供 API 查询，前端页面后续补齐。

## 4. 常见问题

### 4.1 首页显示 API unreachable

说明前端访问不到后端。检查：
- 后端是否已启动（8000）
- 前端代理是否正常（5173）

### 4.2 对话返回占位响应

当前 V0.1 先打通对话/卡片/动作/审计的端到端闭环，智能体能力与业务工具会在后续功能块逐步接入。
