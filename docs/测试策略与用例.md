# DDUP 测试策略与用例（TDD）

## 1. 目标

- 用测试驱动开发（TDD）的方式推进功能：先写用例（Fail）→实现（Pass）→重构（保持 Pass）。
- 用例覆盖产品原型与技术方案的核心闭环：
  - 首页对话（SSE）→ 结果卡 → 动作落地 → 审计可追溯
  - 空间与权限边界（最小 RBAC/对象级）→ 智能体边界
  - 后续：知识图谱、飞书/微信/多维表格联动、统计仪表盘等

## 2. 测试分层

### 2.1 后端（FastAPI）

- 单元测试：模型/工具函数/权限判断（pytest）
- 接口测试：API 请求→响应→数据库副作用（pytest + TestClient）
- 协议测试：SSE 事件序列、卡片 schema、引用结构

### 2.2 前端（React）

- 组件测试：关键页面渲染、交互、请求调用（vitest + testing-library）
- 协议渲染测试：SSE 文本/卡片事件解析（后续拆出纯函数后单测）

## 3. 用例映射（与原型/方案对应）

### 3.1 F1 空间/权限/审计

- BE-SPACE-001：首次访问自动创建个人空间
- BE-SPACE-002：携带 X-Space-Id 时必须是成员，否则 403
- BE-AUDIT-001：执行关键动作写审计（action.execute）
- BE-AUDIT-002：查询审计只返回当前用户+当前空间的数据

### 3.2 F2 对话（SSE）/卡片/动作

- BE-CHAT-001：创建会话成功（绑定 space_id、user_id）
- BE-CHAT-002：SSE 返回包含 message.delta + card.add + done
- BE-ACT-001：actions/execute 返回 ok 且写审计
- FE-CHAT-001：首页对话页可渲染对话区与输入框
- FE-SPACE-001：首页可拉取 spaces 并展示空间选择器

### 3.3 后续（先写用例，先标记为 xfail/skip）

- BE-GRAPH-001：候选实体关系写入待入图队列
- BE-FEISHU-001：写入多维表格受空间外发开关控制
- BE-ANALYTICS-001：统计仪表盘指标聚合口径正确

## 4. 执行与记录

- 执行入口：
  - 后端：`apps/api` 下 `python -m pytest`
  - 前端：`apps/web` 下 `npm test`
- 测试结果记录：
  - 统一记录到 [测试记录.md](./测试记录.md)
  - 每次记录应包含：commit、用例通过/失败、失败原因、是否已记录到阻塞文档
