# DDUP 开发者指南（V0.1）

## 1. 工程结构

- `apps/api/`：FastAPI 后端
- `apps/web/`：React 前端（Ant 风格，适配移动端）
- `infra/`：本地 PostgreSQL（docker-compose）
- `docs/`：文档

## 2. 关键约定

- 请求头：
  - `X-User-Id`：当前用户（开发环境默认 `dev-user`）
  - `X-Space-Id`：当前空间（可选；不传则自动创建并使用个人空间）
- 对话返回：
  - SSE event：`message.delta` / `card.add` / `done` / `error`
  - 结构化卡片用于前端渲染与“可落地动作”
- 审计：
  - 对关键操作写入审计，便于追溯与后续安全治理

## 3. 本地开发

### 3.1 后端

进入 `apps/api/`：
- 安装：`python -m pip install -r requirements.txt`
- 测试：`python -m pytest`
- 启动：`python -m uvicorn app.main:app --reload --port 8000`

### 3.2 前端

进入 `apps/web/`：
- 安装：`npm install`
- 测试：`npm test`
- 启动：`npm run dev`

## 4. 文档如何保持可更新

### 4.1 API 文档（OpenAPI/Swagger）

- 后端启动后访问 Swagger：`/docs`
- OpenAPI JSON：`/openapi.json`
- 导出 OpenAPI 到文档目录：运行 `apps/api/tools/export_openapi.py`

### 4.2 数据库文档（Schema）

数据库文档由 SQLAlchemy 模型元数据自动生成（不依赖实际连接数据库）。
- 生成脚本：`apps/api/tools/generate_db_schema_md.py`
- 生成产物：`docs/generated/db_schema.md`

### 4.3 推荐工作流（强制一致）

每次修改 API 或模型后：
1. 运行测试（后端 pytest、前端 vitest）
2. 运行文档生成脚本（OpenAPI 与 DB schema）
3. 提交本地 git（把生成产物一起提交）

## 5. Git hook（可选）

如果你希望在提交前自动刷新文档：
- 使用 `git config core.hooksPath .githooks`
- Windows PowerShell 可用：`.githooks/pre-commit.ps1`
