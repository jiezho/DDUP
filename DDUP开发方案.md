# DDUP 开发方案（任务分配与编排）（V0.1）

> 用途：依据产品原型与技术方案，拆解为可直接用于“生成代码/分配任务/编排迭代”的开发方案。  
> 约束：后端 Python FastAPI；前端 React（Ant 风格，适配移动端）；数据库 PostgreSQL；智能体以 hermess agent 为基础；对接飞书与微信，数据可与飞书多维表格联动。  
> 交付纪律：每完成一个“完整功能块”，必须 **测试通过** 且 **提交到本地 git**。

## 2. 仓库与工程化约定（建议）

### 2.1 单仓结构（Monorepo）

- `apps/api/`：FastAPI 后端
- `apps/web/`：React 前端（Ant 风格，响应式）
- `packages/shared/`：前后端共享类型（可选：OpenAPI client/TypeScript types）
- `infra/`：docker-compose、SQL 初始化、nginx（可选）
- `docs/`：产品/技术/开发文档（可保留在根目录也可）

### 2.2 分支与提交策略

- 分支建议：`main`（稳定）+ `develop`（日常集成）+ `feat/*`（功能块开发）
- 本地提交要求（每个功能块一次或多次提交均可，但必须在功能块“完成”时留一个可回滚的稳定提交）：
  - `feat(scope): ...`
  - `fix(scope): ...`
  - `test(scope): ...`
  - `chore(scope): ...`
  - 例：`feat(chat): sse streaming with cards schema`

### 2.3 测试要求（最小集合）

- 后端：单元测试 + 接口测试（建议 pytest）
- 前端：组件测试（建议 React Testing Library）+ 关键流程 E2E（建议 Playwright）
- 每个功能块的“完成定义”必须包含：新增/变更 API 的测试用例；关键 UI 的渲染与交互校验；权限边界用例（若该功能块涉及权限）。

## 3. 迭代编排（功能块划分）

说明：
- “功能块”按可独立交付、可测试、可提交来定义；每块都包含：后端、前端、DB 迁移、测试、文档与本地 git 提交。
- 编排顺序按依赖从底座到业务：工程化底座 → 身份与空间 → 对话与卡片协议 → 内容域（待办/术语/资讯）→ 图谱 → 集成 → 统计 → 文件/论文增强。

### 功能块 F0：项目脚手架与基础设施

**目标**
- 建立可运行的后端/前端工程；本地可启动；具备基础 CI 思维（哪怕只在本地运行）。

**后端任务**
- 初始化 FastAPI 项目结构：配置管理、日志、异常处理、健康检查 `/healthz`
- 初始化数据库连接（PostgreSQL）、迁移框架（如 Alembic）

**前端任务**
- 初始化 React + Ant 风格 UI（移动端优先布局）
- 路由与布局：底部 Tab 框架 + 对话页占位 + 全局请求封装

**测试**
- 后端：`/healthz` 接口测试
- 前端：App 渲染测试（路由与底部 Tab）

**完成定义**
- 本地启动后能访问：前端首页 + 后端健康检查

**提交**
- `chore(init): scaffold api and web with db`

### 功能块 F1：认证、空间与权限（RBAC/对象级）+ 审计底座

**目标**
- 支撑“空间绑定（个人/项目/团队）”和“智能体权限边界”，并具备审计记录能力。

**后端任务**
- 数据模型：spaces、members、roles、role_bindings、audit_logs
- API：
  - 空间：创建/切换/成员管理（最小实现可先只做个人空间 + 切换占位）
  - 权限校验中间件：基于 space_id + role
  - 审计写入：对关键操作写审计（登录/读取/写入/外发）

**前端任务**
- 空间选择器（对话页顶部空间下拉）
- 我的页：权限/审计入口页占位（后续 F7/F8/F9 补全）

**测试**
- 后端：空间隔离与权限拒绝用例；审计日志写入用例
- 前端：空间切换 UI 与请求携带 space_id

**完成定义**
- 任意业务 API 必须能在服务端拿到 space_id 并做权限判断（哪怕规则很简化）

**提交**
- `feat(auth): spaces rbac and audit base`

### 功能块 F2：对话会话（SSE）+ 结构化结果卡协议 + 动作执行框架

**目标**
- 实现“首页对话即入口”，后端可流式返回；前端可渲染卡片并触发动作（不依赖全部业务完成）。

**后端任务**
- 会话与消息：chat_sessions、chat_messages、chat_cards、chat_citations、chat_actions
- SSE：
  - event：`message.delta`、`card.add`、`citation.add`、`done`、`error`
- 卡片协议（最小集）：
  - 简报卡（brief）
  - 分析卡（analysis）
  - 术语卡（term）
  - 引用卡（citations）
  - 工具卡（tool_call / tool_result）
- 动作执行框架：
  - `POST /api/actions/execute`：统一入口（入图/转待办/写入多维表格/发飞书等先做占位实现）
- hermess agent runtime 接入（最小占位）：
  - 先实现“无模型模式”：用规则/Mock 生成卡片，保证端到端链路可跑

**前端任务**
- 对话页：
  - ChatThread（消息流渲染）
  - Composer（输入框、发送、取消）
  - QuickActions（快捷指令：简报/统计/入图/今日待处理等先做占位）
  - ResultCardRenderer（按 card.type 渲染）
- 卡片动作：
  - 点击“转待办/写入图谱”等触发 `/api/actions/execute`

**测试**
- 后端：SSE 流式接口测试；卡片 schema 校验测试；actions execute 鉴权测试
- 前端：SSE 消息流渲染测试；卡片渲染快照；动作按钮触发请求测试
- E2E：输入一句话 → 出现至少一张卡片

**完成定义**
- 用户在对话页输入后，能看到流式返回，并至少渲染出一张结构化卡片；卡片动作可调用后端（占位也可）且记录审计。

**提交**
- `feat(chat): sse sessions cards and actions`

### 功能块 F3：助手域（待办/习惯/灵感）+ “转待办”动作落地

**目标**
- 支撑原型中的 C1/C2/C3/C4，并让对话页的“转待办”真正落库可查看。

**后端任务**
- 数据模型：todos、habits、habit_logs、ideas
- API：
  - 待办：CRUD、按日期/状态筛选
  - 习惯：定义、打卡、统计（本周/本月）
  - 灵感：收件箱列表、详情、标签（可选）
- actions：
  - 支持 `action.type = create_todo`、`action.type = create_idea`

**前端任务**
- 助手 Tab：
  - 日历+清单（先做列表版，日历视图可后续增强）
  - 习惯打卡（列表 + 打卡按钮）
  - 灵感收件箱（列表）与详情（抽屉/全屏）
- 对话页动作：
  - 分析卡/建议卡的一键“转待办”

**测试**
- 后端：待办与习惯打卡接口测试；action→落库测试
- 前端：待办列表渲染与状态切换测试；灵感新增与详情打开测试

**完成定义**
- 对话中生成的行动项可以一键落到待办；助手页可查、可勾选、可统计。

**提交**
- `feat(assistant): todos habits ideas with action landing`

### 功能块 F4：学习域（术语库/卡片学习/复习队列）+ “加入学习”动作落地

**目标**
- 支撑原型 B3/B4/B5，且由对话/资讯提炼的术语可进入学习流程。

**后端任务**
- 数据模型：terms、term_reviews（复习队列/间隔复习状态）
- API：
  - 术语库：创建/编辑/列表/搜索
  - 学习卡片：获取今日任务、提交结果（懂了/再来/收藏）
  - 复习队列：按到期时间拉取
- actions：
  - `action.type = add_terms`（批量）

**前端任务**
- 学习 Tab：
  - 术语库（列表/搜索）
  - 卡片学习（懂了/再来/收藏）
  - 复习队列（列表/进入卡片）
- 对话页动作：
  - 术语卡一键“加入学习”

**测试**
- 后端：间隔复习状态迁移测试；批量 add_terms 测试
- 前端：卡片交互测试；学习任务拉取测试

**完成定义**
- 术语可被对话/卡片动作创建并进入学习闭环；学习行为可统计到“统计仪表盘”所需基础指标。

**提交**
- `feat(learning): terms cards and spaced review`

### 功能块 F5：资讯域（资讯源管理/信息流/稍后读）+ “简报/稍后读”落地

**目标**
- 支撑 D1/D2/D3；对话“生成简报”可引用资讯来源；稍后读可落库。

**后端任务**
- 数据模型：feed_sources、feed_items、read_later
- API：
  - 资讯源：CRUD（MVP 可先手工添加 RSS URL/名称）

  - 稍后读：添加/移除/列表
- actions：
  - `action.type = add_read_later`
  - `action.type = generate_brief`（MVP 可先用规则聚合/占位；后续接入 LLM）

**前端任务**
- 资源 Tab：
  - 资讯信息流（列表/未读筛选/收藏/稍后读）
  - 资讯源管理（列表/新增）
  - 稍后读列表
- 对话页：
  - 简报卡支持“写入稍后读/提炼术语/发飞书（占位到 F8）”

**测试**
- 后端：资讯源 CRUD；稍后读状态迁移；简报引用结构校验
- 前端：信息流交互（稍后读、已读）测试

**完成定义**
- 简报卡带 citations（资讯条目 id/链接/时间）；稍后读列表可从对话/信息流双入口写入。

**提交**
- `feat(feeds): sources feed list read-later and brief card`

### 功能块 F6：知识图谱（实体/关系/证据/待入图）+ “入图”动作落地

**目标**
- 支撑 D9/D10/D11；对话输出的候选实体关系可以进入待入图队列并人工确认后入库。

**后端任务**
- 数据模型：graph_entities、graph_relations、graph_evidences、graph_pending
- API：
  - 实体/关系 CRUD（MVP 可先做新增与查询）
  - 待入图队列：列表、审批（通过/驳回）、写入图谱
  - 证据：绑定来源对象（feed_item/paper/file/note）与片段/页码
- actions：
  - `action.type = graph_write_pending`
  - `action.type = graph_commit`

**前端任务**
- 资源 Tab：
  - 知识图谱页：先做“列表 + 邻接关系视图”，关系网画布可后续增强
  - 待入图队列：审核通过/驳回
  - 实体详情：来源证据展开
- 对话页：
  - 候选关系卡一键“写入待入图队列”

**测试**
- 后端：待入图审批流程测试；证据绑定与查询测试；权限边界（图谱写入需显式权限）
- 前端：待入图审批交互测试；实体详情引用展开测试

**完成定义**
- “内容→候选→人工确认→入图→可追溯证据”闭环跑通。

**提交**
- `feat(graph): entities relations evidences and pending workflow`

### 功能块 F7：统计与复盘（仪表盘页面）+ 指标聚合

**目标**
- 支撑 A5（统计仪表盘）与 E2（统计与复盘）；对话分析卡可跳转/引用统计。

**后端任务**
- 指标聚合 API：
  - 学习：完成天数、掌握率、复习完成率
  - 习惯：连续打卡链、周/月热力
  - 阅读：阅读时长、标注数（若 F10 未做则先占位）
- 复盘生成：产生结构化“复盘卡”（可用于飞书/微信分享）

**前端任务**
- 统计仪表盘页（A5）：概览 + 图表占位 + 智能分析区
- 我的-统计与复盘（E2）：历史复盘列表（MVP 可只展示最近）

**测试**
- 后端：指标聚合准确性测试（使用固定数据集）；复盘卡 schema 测试
- 前端：仪表盘渲染测试；筛选（空间/时间范围）测试

**完成定义**
- 仪表盘能展示真实指标（来自 F3/F4/F5/F6 产生的数据）；复盘卡可生成并可用于后续外发。

**提交**
- `feat(analytics): dashboard metrics and review cards`

### 功能块 F8：飞书集成 + 多维表格联动（写入为主）

**目标**
- 支撑 E10/E11；对话/统计/模板产出可写入飞书多维表格并记录审计。

**后端任务**
- 飞书 OAuth 授权与 token 安全存储（最小可用）
- 多维表格：
  - base/表绑定、字段映射、写入策略（默认仅结构化字段 + 引用）
  - 手动同步接口（`/api/integrations/feishu/bitable/sync`）
- 飞书消息推送：
  - 简报/复盘发送到飞书群（卡片消息，MVP 可先文本）
- 审计：外发必须记录（谁、何时、发了什么摘要、目标空间）

**前端任务**
- 我的-集成与连接（E10）：连接/解除、默认动作
- 我的-多维表格联动（E11）：绑定 base/表、字段映射编辑、同步按钮
- 对话页：卡片动作“写入多维表格/发到飞书”

**测试**
- 后端：鉴权与权限（外发开关）测试；写入 payload 结构测试；失败重试/错误返回测试
- 前端：连接状态 UI；写入动作触发与结果提示

**完成定义**
- 从对话或统计页面一键写入多维表格成功；失败有明确提示；审计可追溯。

**提交**
- `feat(integrations): feishu oauth and bitable sync`

- 前端：分享卡预览组件测试

**完成定义**
- 用户可在不暴露原始材料的前提下分享摘要；审计可查。

**提交**
- `feat(share): wechat share cards`
